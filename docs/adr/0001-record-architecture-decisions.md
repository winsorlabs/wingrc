# 1. Record architecture decisions

Date: 2026-06-26
Status: Accepted

## Context
We need a durable record of why the platform is built the way it is, so
contributors (and future us) don't relitigate settled choices.

## Decision
Use lightweight ADRs (one markdown file per decision) committed to the repo.

## Consequences
Every significant technical decision gets a short, dated, immutable record.
